# نظام أوامر التكليف — Task/Assignment Order System

تطبيق لإدارة أوامر التكليف، الإداريات اليومية، التقارير، الإحصائيات، المهام، وبوابة منفذ البرنامج.

## ما تم تجهيزه هنا

- `index.html` — نفس التطبيق الأصلي، مع تعديل بسيط: كان يعتمد على `window.storage` (وهي ميزة خاصة بمعاينة Artifacts داخل claude.ai فقط ولا تعمل خارجها إطلاقًا)، فتم استبدالها بدالتين تتصلان بخادم حقيقي (`/api/state`) لحفظ واسترجاع البيانات.
- `api/state.js` — دالة خادوم (Vercel Function) تحفظ حالة النظام الكاملة في قاعدة بيانات Redis (عبر تكامل Upstash من متجر Vercel)، بحيث تكون البيانات دائمة ومشتركة بين كل من يفتح الرابط.

## خطوات النشر على Vercel (خطوة بخطوة)

### 1. رفع المشروع إلى GitHub
```bash
cd task-order-system
git init
git add .
git commit -m "Initial commit: task order system"
git branch -M main
git remote add origin https://github.com/h3377/task-order-system.git
git push -u origin main
```
(أنشئ المستودع فارغًا أولًا من github.com/new باسم `task-order-system` تحت حسابك h3377.)

### 2. استيراد المشروع في Vercel
1. من https://vercel.com/new اختر مستودع `task-order-system`.
2. Framework: **Other** (لا يوجد أمر بناء مطلوب).
3. اضغط **Deploy**.

### 3. إضافة قاعدة بيانات Redis (ضرورية لحفظ البيانات)
1. داخل المشروع في Vercel: تبويب **Storage** → **Create Database** → **Redis** (Upstash).
2. اربطها بهذا المشروع — سيضيف Vercel متغيرات البيئة تلقائيًا.
3. أعد النشر (Redeploy) بعد الربط.

## ملاحظات مهمة حول هذا النظام تحديدًا
- لا يوجد في هذا التطبيق نظام تسجيل دخول/صلاحيات للموظفين — أي شخص يملك الرابط يستطيع التعديل. بوابة "منفذ البرنامج" فيها حقل دخول باسم المدرب فقط (بدون كلمة مرور)، وهذا مقصود على ما يبدو كتصميم بسيط داخلي وليس نظام حماية.
- إن كان هذا النظام سيُستخدم من قِبل عدة جهات/مدارس بشكل عام (رابط عام)، أنصحك بإضافة حماية دخول (كلمة مرور عامة على الأقل، أو نظام حسابات مثل النظام الآخر) — أخبرني إن رغبت وسأضيفها.
- الحفظ Last-write-wins كما في النظام الآخر.
